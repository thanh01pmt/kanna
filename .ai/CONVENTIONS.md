# Coding Conventions

## Code Style
- Use TypeScript for all source code.
- Import dependencies using ESM syntax.
- Adhere strictly to the tailwind layout styles defined in `DESIGN.md`.

## File Naming
- Components: PascalCase (e.g., `AttachmentPreviewModal.tsx`, `TextMessage.tsx`).
- Utilities & scripts: camelCase/kebab-case (e.g., `pathUtils.ts`, `dev-ports.ts`).
- Tests: `*.test.ts` or `*.test.tsx`.
