# ADR 04: Add Agent Providers Integration (Antigravity & Pi)

## Overview
Successfully integrated two new agent providers into Kanna's architecture: **Antigravity (`agy` CLI)** and **Pi (`pi` CLI)**. These providers utilize a child process JSONL streaming pattern similar to Codex, enabling autonomous orchestration natively within Kanna.

## Implementation Details

### 1. Types & Catalog Expansion
- **File:** `packages/shared/src/types.ts`
- Added `antigravity` and `pi` to the `AgentProvider` type enum.
- Established `ANTIGRAVITY_REASONING_OPTIONS` (`minimal`, Low, Medium, High) and `PI_REASONING_OPTIONS` (Off, Minimal, Low, Medium, High, XHigh).
- Updated `PROVIDERS` catalog with documented defaults (`gemini-3.5-flash` for Antigravity SDK and `gpt-5.5` for Pi Agent).

### 2. Project Discovery Adapters
- **File:** `packages/server/src/discovery.ts`
- Created `AntigravityProjectDiscoveryAdapter` scanning `~/.antigravity/projects/` for JSONL metadata logs.
- Created `PiProjectDiscoveryAdapter` scanning `~/.pi/projects/` mirroring the same extraction logic.
- Registered both in `DEFAULT_PROJECT_DISCOVERY_ADAPTERS`.

### 3. Server Managers (CLI Child Processes)
- **Files:** `packages/server/src/antigravity-app-server.ts`, `packages/server/src/pi-app-server.ts`
- Developed robust streaming adapters that spawn `agy` and `pi` via Node's `child_process.spawn`.
- Implemented real-time JSONL stdout parsing to normalize CLI events (`tool_call`, `tool_result`, `text`, `result`) into Kanna's standard `TranscriptEntry` format via an asynchronous queue system.

### 4. Agent Coordinator Orchestration
- **File:** `packages/server/src/agent.ts`
- Integrated both managers into `AgentCoordinator` state control.
- Updated `startTurnForChat` to construct prompt streams and execute the target CLI based on the resolved active provider (`antigravity` or `pi`).

## OpenSpec Finalization
- Created `openspec/changes/add-agent-providers/` following the Stage 1 proposal framework.
- Completed all tasks across `tasks.md`, satisfying the provider extensibility requirements.
