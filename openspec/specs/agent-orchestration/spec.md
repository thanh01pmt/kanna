## Requirements
### Requirement: Supported Agent Providers
Kanna SHALL support integration with the following agent providers:
- `claude` (In-process SDK)
- `codex` (Child process via JSON-RPC)
- `antigravity` (Child process via JSONL streaming)
- `pi` (Child process via JSONL streaming)

#### Scenario: Provider Selection
- **WHEN** a user starts a chat in a project discovered via Antigravity or Pi
- **THEN** Kanna utilizes the respective CLI runtime (`agy` or `pi`) to stream responses and manage tool calls.

### Requirement: Capturing Pi Reasoning Logs
Kanna SHALL capture `thinking_delta` events generated by the Pi agent stream and append them as `assistant_thinking` entries in the transcript.

#### Scenario: Streaming thinking events
- **WHEN** the Pi agent streams thinking content during its execution turn
- **THEN** the server captures `thinking_delta` events and streams `assistant_thinking` entries to the client.

### Requirement: Rendering Reasoning in UI
The frontend SHALL process `assistant_thinking` entries and render them in a distinct, collapsible panel showing the agent's step-by-step thinking process.

#### Scenario: Displaying collapsible thoughts
- **WHEN** the frontend receives `assistant_thinking` entries in the chat stream
- **THEN** it renders them inside an accordion-style component showing "Pi's Thoughts" or a similar title, which is expanded or collapsed.
